1 ¿Qué hace este apartado?

El apartado Análisis de Fuentes de Noticias permite al usuario ingresar dominios de sitios financieros (por ejemplo bloomberg.com, cnbc.com) y seleccionar una empresa. El sistema visita esas páginas, extrae el contenido visible y, usando inteligencia artificial o un motor léxico, genera una recomendación de inversión consolidada: Comprar, Vender o Mantener.

2 Flujo completo paso a paso

1. Usuario ingresa dominios (bloomberg.com, cnbc.com) + nombre empresa
↓
2. Frontend valida cada URL → GET /api/news/validate-url?url=...
↓
3. Frontend envía análisis → POST /api/news/analyze-sources
Body: { company: "Apple", urls: ["bloomberg.com", "cnbc.com"] }
↓
4. Backend: Hace fetch del HTML de cada dominio (timeout 15 s)
↓
5. Backend: Extrae texto legible, máx. 5,000 caracteres por fuente
↓
6. Backend: Analiza sentimiento (Claude API o lexicón determinista)
↓
7. Backend: Consolida puntajes de todos los dominios
↓
8. Frontend recibe: { verdict: "BUY", score: 0.72, confidence: 0.85, reasoning: "..." }

3 ¿Los datos son reales o demo?

El sistema usa una estrategia multi-proveedor con degradación gradual. Dependiendo de las API keys configuradas en el backend:

Fuente ¿Real o Demo? Condición Uso
Yahoo Finance RSS ✅ Real Siempre activa (sin API key) Noticias del ticker en tiempo real
Finnhub ⚙ Real si hay key FINNHUB_API_KEY en .env Noticias estructuradas, 30 días atrás
NewsAPI ⚙ Real si hay key NEWSAPI_API_KEY en .env Búsqueda de noticias por empresa/símbolo
Polygon ⚙ Real si hay key POLYGON_API_KEY en .env Noticias de ticker específico
Alpha Vantage ⚙ Real si hay key ALPHA_VANTAGE_API_KEY en .env Sentimiento de noticias con puntuación propia
URLs del usuario ✅ Real Siempre activo Scraping HTML del dominio ingresado
Anthropic Claude (IA) ⚙ Real si hay key ANTHROPIC_API_KEY en .env Análisis de sentimiento con LLM
Lexicón determinista ⚠ Fallback Si Claude API no está disponible Conteo de palabras alcistas/bajistas
Templates hardcodeados ⚠ Demo Si TODAS las APIs fallan Último recurso, datos de ejemplo

4 ¿Cómo se calcula BUY / SELL / HOLD?

El score final va de -1.0 (muy bajista) a +1.0 (muy alcista) y se promedia entre todos los dominios analizados:

BUY
Score > 0.3
+ Confianza > 0.4
SELL
Score < −0.3
+ Confianza > 0.4
HOLD
Cualquier otro caso
(valor por defecto conservador)

5 ¿Qué hace exactamente con los dominios ingresados?

Importante: El sistema NO busca noticias específicas dentro del dominio. Toma lo que esté en la página raíz del dominio en ese momento y analiza ese contenido para determinar si es alcista o bajista respecto a la empresa seleccionada.
  1. Hace fetch("https://bloomberg.com") con User-Agent de Chrome y timeout de 15 segundos.
  2. Extrae el texto visible: elimina scripts, estilos y etiquetas HTML.
  3. Toma los primeros 5,000 caracteres del texto resultante.
  4. Construye un objeto pseudo-artículo con el título de la página y ese texto.
  5. Lo envía a Claude API con el prompt: "Analiza este contenido respecto a [empresa]. Responde SOLO con JSON: score, verdict, confidence, reasoning."
  6. Si Claude falla o no hay API key → usa el lexicón determinista (palabras clave en español e inglés).

7 Salvaguardas y disclaimer

Todos los resultados incluyen el siguiente disclaimer constitucional:

"Este análisis de noticias es informativo y no constituye orden ni recomendación ejecutable."

Además, cada fila de señal está marcada con ia_revisada: true para garantizar trazabilidad en la auditoría. El ser humano siempre toma la decisión final.
