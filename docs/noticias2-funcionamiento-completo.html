1 ¿Qué es Noticias 2?

Noticias 2 es el módulo de análisis de sentimiento de noticias financieras del equipo 003-team-02 (core indicadores). Analiza titulares reales obtenidos de Yahoo Finance RSS (y opcionalmente Finnhub, NewsAPI, Polygon) para el ticker activo en la Watchlist, genera un veredicto de inversión (COMPRAR / VENDER / ESPERAR) y lo integra en la Tabla de Confluencias del dashboard.

Diferencia clave vs "Noticias" (otro equipo): Noticias 2 trabaja con datos estructurados de APIs financieras reales, filtra por fuentes seleccionadas por el usuario, genera razonamiento basado en titulares textuales específicos y se integra con el Chat IA mediante prefill del input.

2 Cómo activar el módulo (flujo del usuario)

1
Selecciona un Ticker en la Watchlist Haz clic en SPY, AAPL, NVDA, etc. en el panel lateral izquierdo. El módulo reaccionará a este ticker automáticamente.
2
Activa el chip "Noticias 2" en Cores de Análisis En el Panel de Control · Simulación, dentro de "CORES DE ANÁLISIS", encontrarás el chip Noticias 2 junto a Indicadores, Fundamental, Técnico, Institucional, Noticias, IA. Actívalo.
3
Presiona "Ejecutar Simulación" El módulo no se renderiza en el DOM hasta que la simulación se ejecute. Este es el gate de visibilidad (Bug 1 corregido). Solo entonces aparece el card de Noticias 2 en el body del dashboard.
4
Selecciona las fuentes activas (máx. 5) En el panel izquierdo del card verás 8 fuentes predeterminadas. Selecciona hasta 5. Por defecto vienen activas: Nasdaq, Investing, CNBC. Usa el botón "Limpiar" para deseleccionar todas.
5
Presiona "🔍 Analizar" Primera vez: botón dice "Analizar". El sistema consulta Yahoo Finance RSS (y APIs adicionales si hay keys). Después del primer análisis, el botón cambia a "🔄 Re-analizar".
6
Lee los resultados (Split Card) La columna izquierda muestra la Evidencia Cruda (titulares reales, fuente, snippet, fecha, enlace). La columna derecha muestra el Razonamiento IA con causalidad explícita.
7
Opcional: Enviar al Chat IA El botón "💬 Enviar análisis al Chat IA" precarga el input del chat con un prompt contextual (no lo envía automáticamente). El usuario presiona Enter para continuar el análisis con el copilot.

3 Comportamiento al cambiar de Ticker

Limpieza automática al cambiar ticker: Al seleccionar un ticker diferente en la Watchlist (ej. pasar de AAPL a MSFT), el sistema ejecuta un reset total:
  • El módulo de Noticias 2 se oculta del DOM
  • El flag simulationHasRun vuelve a false
  • Se borra toda la Evidencia Cruda y el Razonamiento IA de memoria
  • El usuario debe ejecutar la simulación nuevamente para el nuevo ticker
Esto previene que datos de Apple aparezcan cuando se analiza Microsoft.

5 Cómo se calcula el veredicto

▲ COMPRAR
Score promedio > +0.25
ó mayoría de artículos alcistas
▼ VENDER
Score promedio < −0.25
ó mayoría de artículos bajistas
— ESPERAR
Score entre −0.25 y +0.25
o señales divididas
MétricaQué significa
ScorePromedio de sentimiento de todos los artículos. Rango −1.0 (muy bajista) a +1.0 (muy alcista). Se muestra como barra visual centrada en neutro.
ConfianzaPorcentaje de acuerdo entre artículos. Alta confianza (>70%) = múltiples fuentes concordantes en la misma dirección.
ArtículosNúmero de artículos reales analizados de las fuentes activas. Mínimo 1 para emitir veredicto.

8 Integración con el Chat IA (Copilot)

El botón "💬 Enviar análisis al Chat IA" NO envía el mensaje automáticamente. Al hacer clic, precarga el input del chat con el siguiente texto contextual:

"Las noticias recientes sugieren [COMPRAR] NVDA (score +0.42, confianza 68%) debido a:
'Aptiv deepens NVIDIA AI partnership — impacto alcista...'.
¿Qué indicadores técnicos actuales respaldan o contradicen esta decisión?"

El usuario puede editar el texto y presionar Enter para enviarlo al Copilot IA.

9 Arquitectura técnica

Frontend (click en chip "Noticias 2" + Ejecutar Simulación)
  ↓ simulationHasRun = true → renderiza <NewsSourcesAnalyzer>
  ↓ usuario selecciona fuentes + presiona "Analizar"

POST /api/news/analyze-sources
  { company: "NVDA", urls: ["nasdaq.com","cnbc.com","bloomberg.com"] }

Backend: routes/news/urlAnalysis.ts
  ↓ resolveProviders(urls) → { yahooFinance, newsapi }
  ↓ fetchNewsData({ symbol: "NVDA", limit: 15 })
      ├── Yahoo Finance RSS (siempre activo, libre)
      ├── Finnhub (si FINNHUB_API_KEY en .env)
      ├── NewsAPI (si NEWSAPI_API_KEY en .env)
      ├── Polygon (si POLYGON_API_KEY en .env)
      └── Alpha Vantage (si ALPHA_VANTAGE_API_KEY en .env)
  ↓ filtro estricto por proveedor activo
      (fallback a Yahoo RSS si filtro da 0 artículos)
  ↓ reasoning = titular real + causalidad explícita

Respuesta:
  { verdict, score, confidence, reasoning, keyPoints, articles[], sourcesNote }

Frontend: NewsSourcesAnalyzer.tsx
  ↓ split card: Evidencia Cruda (izq) + Razonamiento IA (der)
  ↓ noticias2Result → badge en Tabla de Confluencias
  ↓ prefillInput → Chat IA al presionar botón

10 Variables de entorno para máximo rendimiento

VariableFuente que activaSin ella
FINNHUB_API_KEYFinnhub Company NewsSe omite Finnhub; Yahoo RSS suplanta
NEWSAPI_API_KEYNewsAPI (CNBC, Bloomberg, Reuters)Se omite NewsAPI; Yahoo RSS suplanta
POLYGON_API_KEYPolygon Ticker NewsSe omite Polygon
ALPHA_VANTAGE_API_KEYAlpha Vantage News SentimentSe omite Alpha Vantage
ANTHROPIC_API_KEYClaude LLM para razonamientoUsa analizador léxico determinista
Sin API keys: El módulo funciona 100% con Yahoo Finance RSS (gratis). Los artículos serán solo de Yahoo Finance sin importar qué fuente seleccione el usuario, y el razonamiento lo genera el analizador léxico (no Claude). Para producción real, se recomienda al menos FINNHUB_API_KEY + ANTHROPIC_API_KEY.

11 Bugs críticos corregidos (FASE 1)

#BugSolución implementada
1 El módulo se renderizaba antes de ejecutar la simulación Gate simulationHasRun: el componente no existe en DOM hasta que la simulación completa
2 El análisis se disparaba automáticamente al activar fuentes o al ejecutar la simulación Eliminado todo useEffect que disparaba análisis. Solo el botón "Analizar" ejecuta el fetch
3 El botón siempre decía "Re-analizar" sin importar el estado Estado hasAnalyzed: "🔍 Analizar" la primera vez → "🔄 Re-analizar" después
4 Al cambiar de ticker la data vieja (ej. Apple) seguía visible useEffect([selectedSymbol]): oculta módulo + borra toda la data en memoria
5 Bloomberg seleccionado → Yahoo Finance aparecía en Evidencia Cruda sin aviso Filtro estricto + fallback transparente con sourcesNote en el UI
6 Razonamiento IA mostraba "Análisis determinista de N titulares..." Detecta texto determinista y lo descarta; construye reasoning desde titulares reales
